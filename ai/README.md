# PromocoesCOMAER AI Knowledge Base

> Entry point for all AI assistants working on this repository.

---

# Purpose

This directory contains the official knowledge base for AI assistants used during the development of PromocoesCOMAER.

It complements the project source code by providing structured documentation about:

- project objectives
- architecture
- business domain
- coding standards
- database
- Oracle integration
- simulation engine
- development workflow

Every AI assistant should consult this knowledge base before modifying the project.

---

# Golden Rules

Always

✔ Read before coding.

✔ Understand before modifying.

✔ Follow the existing architecture.

✔ Respect business rules.

✔ Preserve consistency.

✔ Keep documentation updated.

---

Never

✘ Invent requirements.

✘ Duplicate business rules.

✘ Ignore AI_RULES.md.

✘ Change architecture without justification.

✘ Introduce technical debt unnecessarily.

---

# Documentation Philosophy

Documentation is considered part of the source code.

Whenever code changes significantly,

documentation should be updated as part of the same work.

---

# Continuous Evolution

This knowledge base is expected to evolve together with the project.

New modules,

new services,

new business rules,

and new architectural decisions

must be documented here.

---

# Final Goal

The objective is that any AI assistant can understand the project architecture, business domain and development standards before writing code.

A correct understanding of the system is more valuable than generating code quickly.

Correct code.

Maintainable code.

Well documented code.

Always.
